# NXGS Rental Platform

NXGS remains a client-only remote-play app. Rental authority lives in Supabase Edge Functions and Postgres RPCs.

## Client Environment

Set these before launching NXGS:

- `NXGS_SUPABASE_URL`
- `NXGS_SUPABASE_ANON_KEY`
- `NXGS_RAZORPAY_KEY_ID`
- `NXGS_TEST_PAYMENT_BYPASS` optional, testing only. Set to `1` to request the backend test-payment bypass before the pricing catalog is loaded.
- `NXGS_RENTAL_FUNCTION_BASE_URL` optional, defaults to `${NXGS_SUPABASE_URL}/functions/v1`

Never set `SUPABASE_SERVICE_ROLE_KEY` or `RAZORPAY_KEY_SECRET` in the client environment.

## Edge Function Secrets

Set these with `supabase secrets set`:

- `SUPABASE_SERVICE_ROLE_KEY`
- `RAZORPAY_KEY_ID`
- `RAZORPAY_KEY_SECRET`
- `CONTROLLER_ADMIN_TOKEN_SECRET` optional, used to sign short-lived Controller admin tokens. If omitted, the service role key is used as the signing secret.

## Controller Admin

The home screen has a Controller tab beside Consoles. Management access is gated by `verifyControllerPin`. Set an operator-chosen access code during deployment; no shared default credential is seeded.

The PIN is stored in `app_settings` under `controller_admin_pin` as a bcrypt hash generated by Postgres `crypt()`, not as plaintext. Successful verification returns a short-lived admin token. The token is required by:

```sql
insert into public.app_settings(key, value)
values ('controller_admin_pin', crypt('<your-access-code>', gen_salt('bf', 10)))
on conflict (key) do update set value = excluded.value;
```

## Console Rows

Each rentable PS5 must exist in `consoles` with:

- `state = 'available'`
- `tailscale_ip`
- `mac_address` or `registered_host_nickname`
- `remote_play_target = 'PS5'`
- Controller metadata can also be saved: `console_pin`, `added_by`, and `added_at`.

`registered_host_nickname` should match the console nickname already registered in NXGS. NXGS uses that local registration and replaces only the host address with the backend-assigned Tailscale IP.

Valid admin-set console states are:

- `available`
- `offline`
- `maintenance`
- `disabled`

Stores are managed separately from consoles. A console is not permanently tied to one store, so store details are not stored on `consoles`.

## Stores, Time Plans, And Pricing

The active customer payment catalog comes from:

- `stores`
- `time_plans`
- `store_time_plan_prices`

The Controller area has separate tabs for Add Store, Manage Stores, Add Time Plans, Manage Time Plans, and Pricing. Time plans define duration in minutes, and pricing links a store to a time plan with an amount in paise.

The client does not hardcode playable durations or prices. `createPaymentOrder` and `extendSession` receive `store_id` and `time_plan_id`, then the backend loads the active price and duration, creates the Razorpay order amount, and calculates session length from `duration_minutes`.

For local testing without Razorpay, enable the backend setting:

- Backend setting: `app_settings.key = 'test_payment_bypass'` and `value = 'true'`

When enabled, the pricing catalog tells the client to request the test-payment path. `createPaymentOrder` and `extendSession` still validate the reservation/session, store, time plan, and pricing on the backend, then mark a test payment as paid through the existing session-confirmation RPCs. `NXGS_TEST_PAYMENT_BYPASS=1` is only a client-side convenience flag and the backend will reject it unless the backend setting is also enabled.

## Scheduled Cleanup

Run these Edge Functions every minute from Supabase scheduled jobs or an external scheduler:

- `releaseExpiredReservations`
- `releaseExpiredSessions`

`releaseExpiredSessions` also handles missing heartbeats. If a session heartbeat is older than 2 minutes, the session becomes `disconnected` and the console is released.

## Realtime

The migration adds `play_sessions` to `supabase_realtime` and grants anon read access to that table so NXGS can subscribe to its active session row. All writes still go through Edge Functions using the service role.

## Deployment

1. Apply `supabase/migrations/202606150001_rental_platform.sql`.
2. Apply follow-up migrations in order.
3. Deploy all functions in `supabase/functions`.
4. Set function secrets.
5. Add or update stores, time plans, and store pricing through Controller admin.
6. Add console inventory through Controller admin or SQL.
7. Launch NXGS with the client environment variables.
